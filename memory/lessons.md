# Lessons - 踩坑记录

## [2026-03-06] QQ Bot 无法用于定时任务
- **问题**: QQ Bot 官方限制主动消息数量，没有用户输入就无法输出
- **影响**: 无法实现定时提醒/日报等功能
- **解决**: 迁移至飞书
- **参考**: https://bot.q.qq.com/wiki/develop/api-v2/server-inter/message/send-receive/send.html
- **标签**: #QQBot #限制 #飞书迁移

## [2026-03-06] 部分行业网站抓取受限
- **问题**: Multilingual.com, Nimdzi, Lionbridge 等因 Cloudflare 或 URL 变更无法直接抓取
- **影响**: 行业监控覆盖不全
- **解决**: 后续配置 Brave Search API 增强覆盖
- **标签**: #爬虫 #Cloudflare #监控

## [2026-03-09] GitHub 分支管理混乱
- **问题**: 同时存在 main 和 master 分支，推送时混淆导致代码不同步
- **影响**: README 更新后网页看不到变更
- **解决**: 统一使用 master 分支，删除多余分支
- **教训**: 一个仓库只保留一个默认分支，避免历史遗留问题
- **标签**: #GitHub #分支管理 #同步

## [2026-03-09] Python subprocess 参数兼容性
- **问题**: `capture_output=True` 在旧版 Python 中不支持
- **影响**: 心跳任务消息发送失败
- **解决**: 改用 `stdout=subprocess.PIPE, stderr=subprocess.PIPE`
- **教训**: cron 环境可能使用系统默认 Python，需保持参数兼容
- **标签**: #Python #subprocess #兼容性

## [2026-03-09] Cron 环境变量缺失
- **问题**: cron 任务运行时找不到 `openclaw` 命令（PATH 不同）
- **影响**: 心跳任务无法发送飞书消息
- **解决**: 使用绝对路径 `/home/admin/.local/share/pnpm/openclaw`
- **教训**: cron 环境与交互式 shell 的 PATH 不同，命令必须用完整路径
- **标签**: #Cron #环境变量 #PATH

## [2026-03-09] 飞书消息缺少 target 参数
- **问题**: `openclaw message send` 需要 `--target` 指定接收者
- **影响**: 消息发送失败
- **解决**: 添加 `--target "ou_adcbc44a6fb7460391e585338f9e1e35"`
- **教训**: 阅读工具文档，确认所有必需参数
- **标签**: #飞书 #消息推送 #参数

## ✅ [2026-03-09] 成功经验：双仓库策略
- **做法**: `openclaw-workspace`（完整备份）+ `openclaw_using_log`（交互日记）
- **效果**: 工作区有完整版本控制，GitHub 有干净的交互记录
- **关键**: evening-sync.sh 分别推送到两个仓库，各司其职
- **标签**: #GitHub #备份策略 #最佳实践

## ✅ [2026-03-09] 成功经验：定时任务输出规范
- **做法**: 信息汇总类任务（新闻/行业动态）按日期分章节追加到固定文档
- **效果**: 避免重复创建文档，历史记录清晰可查
- **关键**: 一个任务对应一个文档，不重复创建
- **标签**: #飞书文档 #定时任务 #输出规范
